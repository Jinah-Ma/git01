VScode(비주얼코드)

노션 링크

이론과 실습

이론과 설정

  • FTP : 인터넷 서비스 중 하나. 웹에서 파일을 주고 받는 것 (파일 확장자로 만든 파일을 업로드, 다운로드 할 수 있게 해주는 프로그램)
  • svg : 웹용 벡터 방식/ 코드로 애니메이션을 만들 수 있다.
  • 폴더 단위로 열어야 된다.
  • 무조건 폴더 안 파일 한 개는 'index.html'로 만들어야 한다. (처음으로 만드는 파일이며, 대표하는 파일이기 때문에.)
  • 운영체제에 따라 대문자와 소문자, /와 ₩(역슬레시)가 다르게 사용되기 때문에 항상 확인해야 한다. (윈도우는 소문자와 /이다.)
  • 형제요소 : 같은 장소에 있는 것 ↔ 호환관계 : 다른 장소에 있는 것
  • 블록요소 : 줄바꿈이 일어난다. 인라인 요소를 포함가능하다. 블록요소는 블록요소를 포함할 수 없다.
  • 인라인요소 : 줄바꿈이 안 된다. 인라인요소는 인라인요소를 포함할 수 있다. 인라인요소는 블록요소를 포함할 수 없다.
  • 엔티티코드 : 공용으로 쓰는 글자 코드로, 우리나라에서 'ㅁ+한자'로 특수문자를 쓰는 것처럼 다른 나라들도 제각각 방식이 다를 수 있기 때문에 공용코드로 써야 되는 것.

작성·설정

  1. 칸은 한 칸만, 파일확장자명의 점도 하나만
  2. 이미지는 복사해서 html폴더에 넣고 프로그램에서 새로고침(F5를 눌러도 되고, 오른쪽 버튼의 새로고침을 선택해도 된다.)하면 나온다.
  3. View에서 word wrap을 체크해준다.(정렬선; 들여쓰기랑 구분해주기 위해 줄로 상태를 보여준다.→나중에 찾기 쉽도록)
  4. 테마설정 ctrl+, 혹은 시스템설정에 들어가서 'workbeach them'을 검색, 원하는 테마로 변경 가능하다.
  5. 오른쪽 메뉴에 'EXTENSIONS'에 들어가서 'live server', 'open in browser'을 검색해 깐다.
  6. ctrl+,(설정창)→emm검색→emmdt : variables를 'ko'로 바꾼 뒤 ok버튼을 누른다. (이렇게 해야 창에서 영어번역창이 뜨지 않는다.)
  7. class(모양 바꾸기), herf(경로) 등 쓸때 AI에 떠서 Enter나 tap을 누르면 문장 전체가 바로 구현이 된다.
  8. 이동경로 (html에서의 '.'(마침표)’는 경로를 뜻한다.)=/ : 한칸 아래, ./ : 지금 여기 (현재 경로; 상위 경로), ../ : 한칸 위로 (../../../ … 반복할수록 1칸씩 폴더를 거슬러 올라간다.)
  9. ':'과 ';'의 차이점 = : - 속성과 값을 나눔(분리), ; - 마무리(마침)
  10. 폴더명이든, 파일명이든 공백을 쓰면 안 된다.

단축키

  • Home : 맨 앞 이동
  • End : 맨 뒤 이동
  • Home+shift 누른 뒤 End를 누르면 블럭이 되어 한꺼번에 지우거나 복사가 가능하다.
  • ctrl+방향키 : 낱말단위로 이동 가능
  • 응용 : ctrl+shift+방향키→낱말단위로 이동하며 블럭을 만든다.
  • ctrl+shift+a : 문장이나 단어를 입력하고 난 뒤 블럭하고 감싸는 태그 넣기
  • ctrl+d : 멀티탭(커서탭)이 여러개가 된다.→감싼 태그를 바꾸고자 할 때
  • ctrl+shift+d : 블록한 것들을 연속으로 복사 붙여넣기 기능
  • ctrl+u : 새창에서 코드를 보여준다.
  • F12 = ctrl+shift+c : 그 창에서 바로 코드를 볼 수 있다. 수정과 다운로드가 가능.(수정의 경우엔 그 창 한정으로, 다시 들어가면 원상복귀 되어있다.)
  • ctrl+, : 설정
  • ctrl+마우스 휠 : zoom
  • -Alt+b : 현재 작업중인 문서 페이지 창을 띄워보여준다.(오른쪽 버튼 눌러보면 'open in default browser'이다.)
  • ctrl+/ : 주석 단축키 (명령어지만 사이트에서 실행이 되지 않고 비주얼코드에서만 보인다.)
  • ctrl+b : 옆 메뉴창 여닫는 단축키
  • shift+alt+r : 현재 파일이 있는 곳의 탐색기(Reveal in file explorer)
  • Alt+Shift+F : 줄맞춤(들여쓰기)→해둬야 찾기도 쉽고, 헷갈리지 않을 수 있다.
  • !+Tab : 기본 내용 바로 완성
  • img+Tab : 이미지 태그 바로 완성
  • ctrl+k+s : 키 설정창
  • ctrl+ait+방향키 : 열 편집
  • ctrl+shift+c : 원하는 곳의 코드 보기(web)

용어

  • head : 머리/ 정보전달(브러우저에게 웹문서의 정보를 전달한다. 보여지지 않는다.)
  • body : 몸/ 내용이 들어가는 곳(사용자에게 웹문서의 정보를 전달한다. 보여준다.)
  • <p> 태그 : 문단, 문장
  • <a> 태그 : 앵커/ class(모양 바꾸기), herf(경로, 링크)
  • style : 다른 언어로 쓰겠다는 것.(무조건 'head'안에서만 쓸 수 있다.)
  • <h> : 제목. h태그는 6단계까지 가능하며 문서의 제목을 의미한다. 남용 금물
  • h1 : 큰 제목
  • h2 : 두번째 제목(작은 제목)
  • target : 타겟
  • blank : 새창열기 (target_blank로 쓴다.)
  • meta : 싱글태그, 닫는 것 없는 단일태그
  • title : 타이틀/ 브라우저에게 문서의 제목(맨 위 탭 부분의 제목)을 알려줌. 검색엔진을 위해 필요하다.
  • br : 단일태그로 줄바꿈 가능함. 문단도 한칸 띄워져 적용이 된다.→블록에서도 사용가능한 태그
  • pre : 띄워쓴대로 보이는 태그. 많이 쓰이진 않는다.
  • address : 웹 하단 주소를 쓰는데 쓰인다.
  •  →메일에 링크를 걸 때 'mailto:메일주소'
  •  →전화번호에 링크를 걸 때 'tel:전화번호'
  •  →메시지에 링크를 걸 때 'sms:전화번호'
