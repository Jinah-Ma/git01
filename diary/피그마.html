피그마

노션 링크

정의

= 프로토타이핑(시안, 테스트 툴)의 오류가 있는지 없는지, 축약했을 때 잘 실행되고, 잘 보이는지 등 사용자 입장에서 테스트 해볼 수 있는 툴

단축키

  • shift키 : 일괄선택/ alt키 : 따로따로 선택가능
  • ctrl+d : 이전 작업을 반복하는 것
  • ctrl+마우스 휠 : 확대축소
  • Rename(ctrl+R) : 이름 일괄 변경(#nn : 01~/ #n : 1~; 오름차순{n}, 내림차순{N} 둘다 동일)
  • ctrl+shift+\(역 슬라이스, enter키 위에) : 왼쪽 창 열고 닫기
  • ctrl+\ : (모든 창 열고 닫기)
  • ctrl+] or [ : 레이어 순서를 위아래로 변경 가능 단축키
  • shift+enter : 하위 레이어에서 실행하면 상위의 것까지 선택된다.
  • ctrl+backspace(←) : 그룹해제

중점

  • 기본 순서는 (오른쪽 설정창) Desige→Prototype(프로토타입) 추가 순
  • 요즘 트렌드 : 가독성↑, 글씨 크게, 심플하게 (글자수는 80자 정도가 적당, 행은 3줄이 적당)
  • 피그마의 글씨파일은 구글 폰트들과 연동되어 있다.(*다른 글씨체를 사용할 경우엔 폰트파일도 함께 보내야 그 글씨체를 구현할 수 있다.)
  • 가독성 좋은 폰트는 고딕(명조는 강조할 글씨 하나 정도만 쓴다.)
  • 기본 폰트 크기는 '16 pt'이며, 기본 8의 배수로 x8하여 사용한다.(정 안 될 경우에도 짝수의 배수를 사용하는 것이 좋다./ 소숫점은 절대X, 드래그로 글씨 크기를 조정하는 것도 마찬가지.)
  • 행간(줄과 줄의 간격) : 글씨 크기*1.15, 1.25, 1.45정도가 가장 좋음(가독성을 따지면 1.75가 더 좋다.)
  • 자간(글자 사이의 간격) : 숫자에 'px'를 무조건 붙여서 쓴다.
  • 문단간격(ex- 이미지 밑 글자와 이미지 간의 간격 등의 황금비율 : 1.618)
  • 피그마는 타 프로그램들과 다르게 좌측 상단이 도화지의 기준
  • 도형은 선택해서 그냥 옮길 수 있지만, 프레임을 옮길 땐 'card'같은 제목부분을 눌러 그걸로 움직여야 한다.(여러 프레임 개체를 선택했을 때에도 마찬가지)

프레임과 그룹의 차이

  1. 프레임 : 자신의 공간으로 들어온 요소만 감싼다. (프레임 범위에서 벗어나면 버린다./ 프레임 공간이 주체, 영역(공간)을 가질 수 있다./ 색상을 가질 수 있고, 스톡도 넣을 수 있다.)
  2. 그룹 : 가지고 있는 레이어가 움직인 곳 만큼 공간이 무한대로 확장된다. (레이어가 주체, 단지 레이어들을 정리하기 위해 하나로 묶어주는 것뿐이다./ 색상·스톡 등은 못 넣음.)
