VScode 태그 및 설명_css

- css는 head의 style에 적용하는 것으로 html을 꾸며준다.

- 원래는 링크를 만들어 적용하는 방식을 써서 해야 되지만 배우는 입장에서 불편하므로 html에 적용하는 방법을 썼다.

  • ★ ctrl+f 검색해서 찾기
    1. css 적용시키는 방법
      • 인터널 방식
        head에 style을 만들어 css를 바로 적용
      • 링크 방식
        head에 link를 걸어 적용
        주로 사용되는 방법 ☆
      • 인라인 방식
        태그 안에 넣어 바로 적용
      • 인포트 방식('@impot')
        링크방식으로 불러온 css문서에 또한번 css파일을 적용시키는 방법
    2. Seclector(선택자)
      1. 태그 선택자
        • 태그를 직접 선택한다.
        • ex- 'p {}'
      2. 아이디 선택자
        • 태그에 id를 부여하고 선택한다.
        • ex- '#menu(부여한 id이름)'
      3. 클래스 선택자
        • 태그에 class를 부여하고 선택한다.
        • ex- '.menu(부여한 class이름)'
      4. 전체 선택자
        • 모든 요소를 선택한다.
        • ex- '* {}'
        • * : asterisk(에스터리스크)
      5. 하위 선택자
        • 요소 밑의 요소를 선택
        • 선택한 요소 전체에게 효과가 적용된다.
        • ex- 'ul li' : 모든 ul의 밑에 있는 li들에게 적용이 된다.
      6. 자식 선택자
        • 요소 밑의 직계자식 하나만 선택
        • ex- '.(부여한 이름)>p' : 선택한 class 밑에 있는 p에만 적용이 된다.
        • *tip) 하위선택자는 모든 하위 요소를 선택하고, 자식선택자는 직접적인 자식 요소만 선택한다.
      7. 인접 선택자
        • 동위관계에 있는 요소 하나만 선택
        • ex- 'ul+p' : 같은 블럭요소로, 동위선상에 있다. ul 바로 밑에 있는 p만 선택해 바꿔준다.
      8. 형제 선택자
        • 모든 동위요소를 선택한다.(절대 위쪽을 선택 할 수 없다.)
        • 같은 부모를 가진 형제관계의 동위요소를 선택할 수 있으며, 밑에 있는 형제요소들에게만 적용이 된다.
        • ex- 'ul~p' : body라는 부모 밑 동위요소인 ul 밑의 p들이 선택되어 적용된다.
        • *tip) 인접선택자와 형제선택자 모두 동위요소에게 적용하지만, 인접은 인접해있는 태그 하나만을 바꿔주고, 형제는 같은 부모 아래 선택된 형제관계의 요소 모두가 적용을 받는다.
      9. 그룹 선택자
        • 여러 요소에 같은 속성을 적용
        • ex- 'h1,p{}'
      10. ☆속성 선택자
        • 속성을 선택해 스타일을 지정
        • (ex) 'a[]' 혹은 '[]' : a=태그선택자, []=속성선택자
          • 차이점 : []안에 적힌 글자가 같다는 가정하에서 'a[]'는 속성값에 []안에 적힌 글자를 포함하는 'a'요소의 것만 선택이 되고, '[]'의 경우엔 []안에 적힌 글자를 포함하는 모든 요소들의 것을 선택하게 된다.
        • 속성의 종류
          1. =(이퀄) : 완전일치 : '[속성=”망고”]'
          2. ^(캐럿) : 시작단어 일치 : [속성^=”망고”]
          3. $(달러) : 끝단어 일치 : [속성$=”망고”]
          4. *(에스터리스크) : 특정단어 포함하는 것 : [속성*=”망고”]
          5. ⇒ 포함만 하고 있으면 적용이 가능하지만, 없으면 타이틀 값이 완전히 일치하는 것만 적용이 된다.
      11. 가상 선택자
        • ':'
        • 하이퍼링크에 스타일을 부여 (둘 이상 사용시 순서가 중요하다./태그의 상태를 선택한다.)
        • <종류>
          1. :link - 링크가 걸린 상태일 때
          2. :visted - 방문했던 링크의 상태일 때
          3. :hover - 마우스가 올라갔을 때/ 가장 많이 쓰인다.
          4. :active - 마우스를 올려 버튼을 계속 누르고 있을 때
          5. input:focus - focus -입력상태로 바뀌었을 때_마우스 이용을 못 할 때 접근성을 위해서
          6. ★
          7. :before - 가상선택자 부분(맨 앞)이 선택된다./ 미처 넣지 못한 컨텐츠를 넣고 싶을 때, 태그 안에 자식으로 추가한다.
            ex- p:before{content: ;}
          8. :after - 마찬가지지만, 맨 뒤에 넣을 수 있다. (참고-이 두가지로 넣은 콘텐츠는 마우스로 긁히지 않는다.)
      12. 가상요소 선택자
        • :nth-child(n)
          →부모의 모든 자식요소(하위요소) 중 n번째
          • 전체 중 n번째 있는 자식을 선택
          • ex- div:nth-child(2) → div에 속해있는 자식 중 2번째 아이를 선택
          • ex- div>p:nth-child(2) → div에 속해있는 자식 p가 2번째에 있을 때만 선택이 가능. 3번째에 있으면 적용이 되지 않는다.
          • :nth-child(odd) →홀수번째 애들만 선택
          • :nth-child(even) →짝수번째 애들만 선택
          • :nth-child(1n{2n, 3n...}) →1n 한번 건너뛰어서/ 2n 두번 건너뛰어서/ 세번 건너뛰어서 등
        • :nth-of-type(n)
          →부모의 자식 중 특정요소의 n번째
          • 전체가 아닌 자식을 특정지어서 그 중 순서가 첫번째 인 것
          • ex- span:nth-of-type(2) → span중에 두번째 span을 선택한다.(span이 여러개일 경우엔 두번째에 있는 span들 모두에게 적용이 된다.)
        • box자식 중 첫번째 p요소에 마우스 호버시 인접한 span의 배경색 변경
          →.box>p:nth-of-type(1):hover+span{background-color: blue;}
    3. 우선순위
      • 캐스캐이딩(cascading_적용순서_우선순위)
        • 적용 순서 방법 : 우선순위가 낮은 것부터 적용시키고, 그 뒤에 순위도를 올리면서 작업을 해야 나중에 값을 수정하기에 용이해진다.
        • → 우선 순위가 하위라도 밑에 쓰면 그게 또 적용이 되어 버리는 경우가 있므로(컴퓨터는 위에서 아래로 한 줄씩 읽기 때문에) 웹페이지의 개발자모드(ctrl+shift+c)를 보면서 하자.
        1. 점수 : inline(1000)/ id(100)/ class, 속성, 가상선택자(10)/ 요소선택자-태그-(1)
        2. 점수를 올리기 위해 같이 쓰는 방법
          ex→ 'body#bgPage1.bgPage2' (공백을 주면 '하위선택자'를 뜻하게 되므로 공백없이 함께 쓴다.)
        3. 선언순서에 의해 하위에 쓰는 방법
          → 동일한 중요도와 특정도를 가진 규칙들 중에서 나중에 선언된 규칙이 우선된다.
        4. 참고
        5. !important : 제일 우선으로 적용되는 아이로, 제일 나중에 하다하다 정 안 될 때 쓰는 것.(수정하기 어렵다.)
    4. inherit_상속
      • 부모 요소의 스타일이 자식 요소로 전달되는 개념
      • ex- border: inherit; -상속제어 태그(활성화 시킨다.)
      • 상속되는 속성 : font-family, font-size, color, line-height 등의 텍스트 관련 속성
      • 상속되지 않는 속성 : border, margin, padding, width, height 등의 박스 모델과 관련된 속성은 기본적으로 상속되지 않습니다. 이러한 속성은 자식 요소에 직접 적용
    5. 글자
      • font-family : 글씨체 변경
      • ex- font-family: "맑음 고딕",바탕,"Times New Roman";
         → 맑음고딕이 없으면 바탕으로, 바탕도 없으면 그 다음 글씨체로 적용이 된다.(순위별로 적용)
         → ","를 하고 안 하고의 차이 : 띄어쓰기. 값 없이 문자가 있다라고 생각하게끔, 폰트 이름이라고 생각 할 수 있도록 하기 위함.
         → 또한 되도록이면 폰트들은 영문으로 쓰는 것이 좋다. 적용이 안 될 수 있기 때문에
      • "san serif" : 고딕체
      • "serif" : 명조체
      •  →두가지 체가 폭이 다르고, 맥과 윈도우는 글씨체가 호환이 되지 않아서 레이아웃이 달라질 수 있으므로 주의해야 한다. 꼭 써야 된다.
      • font-style: italic과 normal 등으로 바꿀 수 있다.
      • line-height : 행간
        ex) line-height: 1.5;
         →글씨크기의 1.5배(=150%), 혹은 px를 붙여 절대값으로 지정할 수도 있다.
         →포토샵 등은 행간이 맨 아래 있지만, css의 행간은 중간에 있다./ 그래서 만약 행간을 50px 주었을 경우엔 상하로 25씩 나눠 가진다.
        ☆블럭의 경우 높이가 있어야 수직정렬이 된다.(행간으로 인해 그렇게 보이는 거지, 막상 줄이 길어지면 밑으로 내려와 밑 컨텐츠와 겹쳐 보여져 버린다. 따라서 한 줄, 적은 단어일 때만 쓴다.)
      • font-weight: 굵기
      • font-size: 2em;
         →글씨 크기 : 1em=16px{em-상대단위}
         →1em=16px, 0.5=8px/ 1.5em=24px=>상대단위라 보여지는 크기가 바뀐다. ★em은 부모 글씨 크기 기준으로 바뀐다. 부모ul의 크기가 12px이고, 1em=12px, 0.5em은 6px이 되어, 12+6=18px가 되어 보여진다.
         →
      • font-size: 1.5rem;
         →rootem=rem/ 루트(뿌리; 최상위)가 기준(즉 html이 기준이 되어 계산되 적용된다./ 24+12=36)
      • font-size: 5vw; or 5vh;
         →브라우저의 "폭"에 맞게 조절이 되어 보여진다.(or 뷰포트의 위드;viewport width) 5vh로 바꾸면 높이에 반응한다.
      • font-variant: small-caps;
         →소문자 크기의 대문자로 만들어준다.(한글은 당연히 안 됨)
      • letter-spacing : 글자와 글자 간격/ 자간→한글의 경우 -0.5px를 주는 게 좋지만 절대적인 건 아니다.
      • word-spacing : 단어와 단어 간격(잘 쓰지 않는다.)/ 어간
      • text-decoration
         →밑줄 속성
         →underline(아래쪽 줄이 생김), overline(위쪽으로 줄이 생김), line-through(취소선이 생김), none(밑줄 없애기)
      • text-transform
         →대소문자
         →uppercase(전부 대문자), lowercase(전부 소문자), capitalize(앞글자만 대문자)
      • 글자의 그림자 지정
        • ex) text-shadow: 1px 1px 2px pink;
           →순서: x축 y축 blur color
           →블러를 넣기 싫으면 쓰는 걸 생략하면 된다.
           →여러개 적용 예시) text-shadow: 1px 1px 2px red, 0 0 1em blue, 0 0 0.2em blue;
    6. 폰트 사이트에 적용하기 방법
      1. 웹사이트 것을 적용시키기
         <구글폰트 적용시키기>
        1. 구글의 무료 폰트 사이트에 들어가 원하는 폰트를 선택한다.
        2. 원하는 폰트 굵기 옆 select'+'버튼을 누르고 나온 창의 소스(위의 것)를 head에 붙여 넣는다.
        3.  →html의 경우 head에 'link'소스를 붙여넣거나
           →css를 만들어 'link'소스를 붙여넣어 적용한다.
        4. 문제점 : 호환성으로 인해, 혹은 구글에서 폰트소스를 내려버리는 경우에 내가 적용해둔 폰트를 못 보게 되는 경우가 생긴다.
      2. 다운로드 받은 폰트를 적용시키기
        1.  <일반 방식>
           →html이나 css에 다운받은 폰트를 적용한다.
           →ex) body{font-family: 'himayari', sans-serif;}
          문제점: 해당 폰트를 함께 보내주어야 한다.(깔려있지 않을 수 있기 때문에)→번거로워진다.
        2.  <@import(임포트)방식>
          ex) @font-face {font-family: 'himayari';
          src: url('./Fonts_Jikjisoft/J-Bold.otf');
          font-weight: 900;
           →내 컴퓨터에 설치해 둔 폰트를 적용시켜 웹사이트를 이용한 사람들이 볼 수 있게 하는 방식
           →다양한 스타일들을 적용시킬 수 있다.
           →번거롭지 않게 사용자가 적용시킨 폰트를 볼 수 있다.
           →단, 글꼴이 호환성으로 인해 보이지 않을 경우가 있으므로 아래 하단에 일반방식으로 윈도우에 일반적으로 들어있는 공용폰트를 여러개 적어 두어야 한다.
        3. tip*)
           →ttf파일이 가장 사용하기 좋다.(윈도우와 맥 상관없이 만들어졌기 때문에)
           →간단한 파일을 만들경우에 만약 사용한 폰트에 제약이 있거나 번거로움을 피하고 싶다면 화면에 구현한 것을 이미지 파일로 저장하여 건넨다.
    7. 링크를 통한 무료 아이콘 집어넣기
      1. 이미지로 내려받아 집어넣을 수도 있지만, 링크로 웹사이트에서 무료로 배포하는 것을 소스로 집어넣을 수 있다.(구글)
      2. 원하는 아이콘을 선택하여 나온 페이지에서 세번째 칸에 있는 소스를 head에 복사붙여넣기 한다. 이후 네번째 칸에 있는 것을 body의 적용하고 싶은 곳에 복사붙여넣기를 하고, 'span'과 '/span'사이에 맨 마지막 cord point를 넣는다.
    8. list-style(리스트 스타일)
      - 세부 속성들 스타일의 축약어(타입, 포지션, 이미지)
    9. 초기화 값
      • font-size: initial; - 폰트사이즈 초기화(initial : 초기화 태그)
      • font-family - 글씨체
        ex - font-family: "맑은 고딕", 돋움, arial;
      • list-style:none - ul, ol의 li 땡땡이와 숫자가 사라짐
         →tip*) body색을 바꾸면 땡땡이의 색이 바뀐다.(+'@'는 바뀌지 않는데, 이건 특정을 해서 바꿔줘야 한다.=a 색상을 바꾸려면 a를 특정해서 바꿔야 한다.)
      • text-decoration: none; - 링크의 밑줄이 사라짐
      • margin:0;/ padding:0;/ border: 0;(1)// 폰트셋팅(2)// 땡땡이와 밑줄 없애기(3)//⇒기본으로 깔고 가야 하는 리셋★
      • overflow-y: scroll; - 스크롤바 생기게 하기(짧은 곳에서 긴곳으로 갈 때 레이아웃이 흔들리지 않게끔 만든다.)
      • box-sizing: border-box; - 박스 크기 안에 보더와 패딩 등이 비율로 알맞게 들어간다.
    10. 컬러
      1. background-color: red;
      2. background-color: rgb(52, 27, 133)
          → 맨 위의 것은 정확한 값이 아니기 때문에 오해가 생길 수 있어, 이왕이면 숫자로 쓰는 것이 좋다.

      • flex
        • ★꼭 부모한테 선언! 그러면 자식들이 배치.
        • 부모는 컨테이너, 자식은 아이템박스라 부른다.
        • dispaly는 상속이 되지 않는데, flex는 바로 밑 하위에만 적용이 된다.
        • (*tip : 반응형을 할 때는 페이지 창을 넓게 보고 작업한다.)
        • 플렉스 안에서는 자동적으로 글자 크기가 줄어든다. 원래는 넘쳐 흐르는게 정상.
        • <flex-direction : 방향설정>
          • flex-direction: row; -기본값 가로(좌->우)
          • flex-direction: row-reverse; -우-> 좌/ 순서도 바뛴다. 자신의 공간은 남겨 둔 채 맨 뒤로 간다.
          • flex-direction: column; -상-> 하
          • flex-direction: column-reverse; -하-> 상
        • <flex-wrap : 컨테이너 안에서 줄바꿈 혹은 정돈>
          • flex-wrap: nowrap; - 박스가 컨테이너의 공간보다 많을때 노랩을 하면 아이템박스를 크기를 줄이며 구겨 집어넣어 쭉 나열
          • flex-wrap: wrap; -랩을 하면 아이템박스의 크기를 고정시킨채로 줄바꿈시켜 정돈
          • flex-wrap: wrap-reverse; -랩리버스를 하면 순서가 바뀐다. (모바일페이지에서 중요한 컨텐츠를 앞으로 배치하고 싶을 때 쓴다.)

          • flex-flow: column wrap; → 디렉션과 랩을 한꺼번에 쓸 수 있는 축약어
        • <justify-content : 주축 정렬>
          • justify-content: flex-start; -기본값/ 주축의 시작점(가로방향이 주축이고, 세로방향이 교차축(반대로 아래로 떨어지는 경우엔 세로방향이 주축이고, 가로방향이 교차축이 된다.))
          • justify-content: flex-end; -오른쪽으로 이동함/ 주축의 끝점에 위치함
          • justify-content: center; -주축 중앙
          • justify-content: space-between; -양쪽 정렬/ 가로축 주축으로 부모의 공간 전체 맞춤으로 간격맞춤(첫번째 위치한 아이 기준으로 끝점까지 균등하게 늘어남./⇒시작점과 끝점 배치 후 중간 박스는 균등배치)
          • justify-content: space-around; -모두 균등배치/ 1번의 좌우여백, 2번의 좌우여백이 병합없이 균등하게 배치된다.
          • justify-content: space-evenly; -1번의 좌우여백과 2번의 좌우여백이 병합되면서 균등하게 배치된다. ⇒전부 균등배치이나 겹치는 부분 병합
        • <align-items : 교차축 정렬>
          • align-items: flex-start; -교차축 시작점 배치/ y축 위에 붙어있다.
          • align-items: flex-end; -교차축 끝점 배치/ y축 맨 밑에 붙어있다.
          • align-items: center; -교차축 중앙정렬/ 수직중앙정렬
          • align-items: baseline; -문자 기준선에 맞춰 배치/ line-heigth행간이 위로가고 그것에 맞춰 정렬되어 있다.
          • align-items: stretch; -교차축에 꽉 차게 늘임(기본 속성으로 늘어나 있는 것-height값-)
        • <align-self : 한 줄 일 때 쓰는 아이/ 교차축>
          • align-self: flex-start; -교차축의 시작지점으로 정렬/ y축 위에 붙어있다.
          • align-self: stretch; -쭉 늘어나는 아이(기본 속성으로 늘어나 있는 것-height값-)
          • align-items와 차이 : 선언을 누구에게 하느냐./ 얼라인셀프는 자식-아이템박스에-한테 직접, 아이템스는 부모에게 선언
        • <align-content: 여러 줄 일 때 쓰는 아이/ 교차축>
          • align-content: flex-start; -시작지점 일치
          • align-content:flex-end; -끝점 일치
          • align-content: space-between; -시작, 끝점 배치 후 중간은 균등배치
          • align-content: space-around; -몽땅 균등배치
          • align-content: stretch; -늘리기

          • justify-content: center;/ align-content: center; -주축과 교차축 모두 센터
